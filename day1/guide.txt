PART 1 — Opening a File ////////////////////////////////////////////////////////////////

Core function:

open(filename, mode)

Example:

f = open("data.txt", "r")

Modes:

Mode	Meaning
"r"	read
"w"	write (overwrite)
"a"	append
"x"	create new file
"b"	binary
"t"	text (default)
 Important Reality
open("file.txt", "r")

 crashes if file doesn’t exist

Always Close File
f.close()

If you forget → memory/resource leak.

Professional Way (Context Manager)
with open("data.txt", "r") as f:
    data = f.read()

Auto closes file.

Always use this.

What strip() does

strip() removes extra whitespace from the beginning and end of a string:

spaces " "
tabs \t
newline \n
Example:
line = "Hello\n"
print(line)          # prints with extra blank line
print(line.strip())  

cursor ///////////////////////////
What happens with read()
with open("test.txt", "r") as f:
    print(f.read())
    print(f.read())
 Step by step:
First f.read()
→ reads everything
→ cursor moves to end of file
Second f.read()
→ cursor is already at the end
→ nothing left to read
→ returns "" (empty)
 That’s why you see:
Hello World

(empty output)
 Fix: reset the pointer
with open("test.txt", "r") as f:
    print(f.read())
    f.seek(0)   #  move cursor back to start
    print(f.read())

Now both prints will show the content.

 What seek(0) means
seek(0) → go to start of file
seek(n) → go to position n (character index)
Example:
with open("test.txt", "r") as f:
    f.seek(6)
    print(f.read())

If file is:

Hello World

Output:

World
 Bonus: check current position
print(f.tell())
tell() → tells current cursor position
 Summary
File has a hidden cursor
read() moves cursor forward
Once at end → nothing left
seek(0) → resets to beginning


error and file handling both : //////////////////////////////////////////
very Important to understand 
1. Two completely different systems
 if / else = your decision logic

You control it.

if file exists:
    do something
else:
    do something else

 No errors are involved here.

 try / except = Python’s error system

Python controls it.

try:
    risky code
except:
    runs ONLY if Python crashes
 2. Your current code (what actually happens)
filepath = Path(r"...\tells.txt")

try:
    if filepath.exists():
        with open(filepath, "w") as f:
            f.write("HI")
        with open(filepath, "r") as f:
            print(f.read())
except Exception as e:
    print("file not found", e)
 Case A: file DOES exist
Flow:
try starts 
filepath.exists() → True 
file opens 
code runs normally 
 NO error happens → except is NOT used

 Result:

Program works
except is ignored
 Case B: file does NOT exist
Flow:
try starts 
filepath.exists() → False 
if block is skipped completely
NOTHING happens inside try
 still NO error
except is NOT triggered

 Result:

No message from except
No error thrown
 KEY INSIGHT (this is your confusion point)

except only runs if Python throws an ERROR

But here:

if filepath.exists():

 You are preventing errors BEFORE they happen

So Python never gets a chance to “crash”

 3. When DOES except actually run?

Only when Python faces a real failure:

Example 1: wrong file open
with open("missing.txt", "r") as f:
    f.read()

 This throws:

FileNotFoundError

 Now except runs

Example 2: division error
10 / 0

 ZeroDivisionError → caught by except

 4. Correct mental model
Think like this:
try:
    "I will attempt this code"
except:
    "If it crashes, I will handle it"

NOT:

try:
    "I will check conditions using if"
 5. Your current pattern = mixing two systems

You are doing:

✔ manual check (exists())
 AND error handling (try/except)

This often leads to confusion.

 6. CLEAN ways to write your program
 Option 1 (NO try/except needed)

If you are already checking existence:

if filepath.exists():
    with open(filepath, "r") as f:
        print(f.read())
else:
    print("File not found")

 Simple logic, no exceptions involved

 Option 2 (PROPER exception usage — recommended)

Let Python handle it:

try:
    with open(filepath, "r") as f:
        print(f.read())

except FileNotFoundError:
    print("File not found")

 This is real-world style

 Option 3 (advanced understanding)
try:
    risky operation
except FileNotFoundError:
    handle missing file
except PermissionError:
    handle access issues
except Exception as e:
    handle unexpected errors

 This is professional-grade handling

 7. Final mental clarity
Situation	What happens
if exists()	no error system involved
try block	Python watches for crashes
except	runs ONLY if crash happens
your current code	prevents errors → so except never run



w3schools //////////////////////////

"r" - Read - Default value. Opens a file for reading, error if the file does not exist
"a" - Append - Opens a file for appending, creates the file if it does not exist
"w" - Write - Opens a file for writing, creates the file if it does not exist
"x" - Create - Creates the specified file, returns an error if the file exists

To open a file for reading it is enough to specify the name of the file:
f = open("demofile.txt")
The code above is the same as:

f = open("demofile.txt", "rt")
Because "r" for read, and "t" for text are the default values, you do not need to specify them.

removal of files and folder /////////////

os.remove(file) it only removes the file that exists otherwise throws an error 
os.rmdir(folder)

but you can remove the folder only when its empty , if it has files already then it will throw an error 
Using the os module, you cannot remove a folder if it contains files.
 Why?
The function:

os.rmdir("folder_name")

only works on empty folders. If the folder has even one file inside, it will throw an error.
 Example (will fail if folder is not empty):
os.rmdir("images_backup")
 How to remove a folder WITH files:
You need to use shutil instead:
import shutil
shutil.rmtree("images_backup")
This deletes:

the folder
all files inside it
all subfolders

 Quick summary:
os.rmdir() → only empty folders
shutil.rmtree() → folders with content
